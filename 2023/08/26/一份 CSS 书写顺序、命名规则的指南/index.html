CSS 书写顺序

位置属性 position、top、right、z-index、display、float 等
大小 width、height、padding、margin)
文字系列 font、line-height、letter-spacing、color-text-align 等
背景 background、border 等
其他 animation、transition 等

不推荐

.example {
    color: red;
    z-index: -1;
    background-color: #9e0;
    display: inline-block;
    font-size: 1.5em;
}

推荐

.example {
    z-index: -1;
    display: inline-block;
    font-size: 1.5em;
    color: red;
    background-color: #9e0;
}

CSS 书写规范

使用 CSS 缩写属性

CSS 有些属性是可以缩写的，比如 padding、margin、font 等等，这样精简代码同时又能提高用户的阅读体验。

不推荐

.list-box {
    border-top-style: none;
    font-family: serif;
    font-size: 100%;
    line-height: 1.6;
    padding-bottom: 2em;
    padding-left: lem;
    padding-right: lem;
    padding-top: 0;
}

推荐

.list-box {
    border-top: 0;
    font: 100%/1.6 serif;
    padding: 0 lem 2em;
}

去掉小数点前的 0

不推荐

font-size: 0.8em;

推荐

font-size: .8em;

简写命名

很多用户都喜欢简写类名，但前提是要让人看懂你的命名才能简写哦！

不推荐

#navigation {
    margin: 0 0 lem 2em;
}
.atr {
    color: #93c;
}

推荐

#nav {
    margin: 0 0 lem 2em;
}
.author {
    color: #93c;
}

16 进制颜色代码缩写

有些颜色代码是可以缩写的，我们就尽量缩写吧，提高用户体验为主。

不推荐

color: #eebbcc;

推荐

color: #ebc;

连字符 CSS 选择器命名规范

长名称或词组可以使用中横线来为选择器命名。
不建议使用 _ 下划线来命名 CSS 选择器，为什么呢？
*输入的时候少按一个 shift 键；
*浏览器兼容问题 （比如使用 _tips 的选择器命名，在 IE6 是无效的）
*能良好区分 JavaScript 变量命名（JS 变量命名是用 _）

这里有一篇破折号与下划线的详细讨论，
英文：点击查看
中文：点击查看

不要随意使用 Id

id 在 JS 是唯一的，不能多次使用，而使用 class 类选择器却可以重复使用，另外 id 的优先级优先与 class，所以 id 应该按需使用，而不能滥用。

不推荐

#info-title {
    font-size: 8em;
}

推荐

.info-title {
    font-size: 8em;
}

为选择器添加状态前缀

有时候可以给选择器添加一个表示状态的前缀，让语义更明了，比如下图是添加了 .is- 前缀。

不推荐

#withdrawal {
    background-color: #ccc;
}

推荐

.is-withdrawal {
    background-color: #ccc;
}

注意事项

1.一律小写
2.尽量用英文
3.不加中槓和下划线
4.尽量不缩写，除非一看就明白的单词
