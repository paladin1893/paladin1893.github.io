Docker方式部署禅道

拉取镜像

docker pull hub.zentao.net/app/zentao:latest

安装镜像

sudo docker run -d \
    --name zentao \
    --restart always \
    -p 9880:80 \
    -p 9806:3306 \
    -v /home/zentao/zentaopms:/www/zentaopms \
    -v /home/zentao/mysqldata:/var/lib/mysql \
    hub.zentao.net/app/zentao:latest

访问ip:9880，安装禅道，mysql默认密码：123456

如果需要修改镜像安装参数，先停止禅道容器，再删除容器，然后重新安装镜像

停止禅道容器

sudo docker stop zentao

删除容器

sudo docker rm zentao

Installing with Docker

安装命令

docker pull owncloud
docker pull mariadb

docker run --name mariadb -e MYSQL_ROOT_PASSWORD=owncloud -v /home/owncloud/mariadb:/var/lib/mysql --restart=always -d mariadb
docker run --name owncloud -p 10080:80 -v /home/owncloud/owncloud:/var/www/html/data --link mariadb:mysql --restart=always -d owncloud

信任域问题

find / -name config.php
编辑 …/config/config.php文件

A typical configuration may look like this:

'trusted_domains' => [
   0 => 'localhost',
   1 => 'server1.example.com',
   2 => '192.168.1.50',
],

仿照格式添加信任域，然后重新访问 http://公网IP:端口，问题解决了

数据目录（默认值，不改）

最新docker部署gitlab指南

2023-07-28
部署运维

参考文档

极狐GitLab Docker 镜像

拉取镜像

sudo docker pull gitlab/gitlab-ce:latest

安装容器

sudo docker run --detach \
  --hostname 192.168.10.33 \
  --publish 9443:443 --publish 9980:80 --publish 9822:22 \
  --name gitlab \
  --restart always \
  --volume /home/gitlab/config:/etc/gitlab \
  --volume /home/gitlab/logs:/var/log/gitlab \
  --volume /home/gitlab/data:/var/opt/gitlab \
  gitlab/gitlab-ce:latest

查看日志

，可以用容器名，也可以用容器id

sudo docker logs -f gitlab

进入容器

sudo docker exec -it gitlab /bin/bash

修改root密码

  1. 切换到相应路径下: cd /opt/gitlab/bin/
  2. 打开控制台: gitlab-rails console -e production
  3. 查询用户账号信息并赋值给u: u=User.where(id:1).first
  4. 设置该用户名的密码: u.password=’Demo9999’
  5. 确认密码: u.password_confirmation=’Demo9999’
  6. 保存信息： u.save
  7. 退出: exit

修改外部访问地址

vi /etc/gitlab/gitlab.rb

external_url 'http://192.168.10.33:9980'
gitlab_rails['gitlab_ssh_host']='192.168.10.33'
gitlab_rails['gitlab_shell_ssh_port']='9822'
gitlab_rails['time_zone'] = 'Asia/Shanghai'

最新centos部署docker指南

2023-07-27
部署运维

2023年7月之后，Docker Compose V1不再更新，与后续新版Docker Destop也不再适配。
取而代之，当前支持的Docker Desktop均使用Docker Compose V2。

参考文档：

Install Docker Engine on CentOS
Overview of installing Docker Compose

安装命令

整理如下：

sudo yum remove docker \
                  docker-client \
                  docker-client-latest \
                  docker-common \
                  docker-latest \
                  docker-latest-logrotate \
                  docker-logrotate \
                  docker-engine

sudo yum install -y yum-utils
sudo yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
sudo yum-config-manager --add-repo http://mirrors.aliyun.com/docker-ce/linux/centos/docker-ce.repo

sudo yum install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

查看版本：

sudo docker -v
sudo docker compose version

启动docker

sudo systemctl start docker
sudo systemctl enable docker

测试docker

sudo docker run hello-world

查看docker容器

sudo docker ps
sudo docker ps | grep zentao

查看docker容器日志，XXXXXX可以是容器名字，也可以是容器id

docker logs -f XXXXXX

进入容器操作

sudo docker exec -it gitlab /bin/bash
